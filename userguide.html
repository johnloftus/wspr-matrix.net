User Guide

Basic Workflow

  1. Open WSPR Matrix.
  2. Enter the reporter callsign, UTC date, and local UTC offset if local time is wanted beside UTC.
  3. Choose Path 1 and Path 2 destination areas using continent, city, and radius.
  4. Build from live ClickHouse data, or use Load WSPRnet Data and Build Matrix to build from WSPRnet JSON data.
  5. Choose the counting mode: Raw Spots, Unique Beacons Per Bin, or Cumulative Unique Beacons.
  6. Use Save Collected Data if you want to keep the collected JSON data for later review.
  7. Review the path matrices and the all-destinations matrix.
  8. Export CSV or print/save PDF from the browser.

Counting Modes

Raw Spots counts every imported report row.

Unique Beacons Per Bin counts each beacon callsign once within each 30-minute band/destination cell.

Cumulative Unique Beacons counts each beacon callsign once across the UTC day and shows the running total by time bin.

Important Notes

  • WSPRnet JSON data should contain headers and rows, or an array of spot objects with reporter, date, band/frequency, grid, and callsign fields.
  • Blank cells mean no matching observed reports in the imported data, not proof that propagation was closed.
  • Endpoint radius controls which beacon grids are included in each path area.
